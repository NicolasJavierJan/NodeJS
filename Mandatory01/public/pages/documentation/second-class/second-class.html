Strict mode:

Strict mode catches problems with the code. Makes it easier to write "secure" JavaScript:

"use strict"

Functions:

Because of hoisting, we can call functions that have not yet been declared. Functions are also fist class citizens in JavaScript: we can pass them as parameters to other functions.

There are different ways to write functions:

Named functions:

function log(message) {

console.log(message)

};

Anonymous functions: Functions without a name.

const randomAnonymous = function (message) {

console.log(message)

};

Arrow functions:

const randomArrowFunction = (message) => console.log(message);

Callback function:

A callback is a function passed as an argument to another function.

Package.json

The package.json file contains data and metadata about a project, such as: name, version and dependencies. It is similar to the "pom.xml" file that Maven uses.

We can create the package.json file on your own or we can use the help of NPM to do it:

npm init

After the package.json file is created, we can add/remove dependencies and install it:

npm install

NPM Install will download packages, folders and code from the dependencies we added to the package.json file. We can also use NPM uninstall to uninstall them.

Express:

Express is a micro-framework, easy to set up. It will help us to create a server and serve an API and webpages.

To use install in our project, we can manually add it in the package.json file by adding it in the dependencies. We can also use

npm install express --save

We then create an app.js file, and start setting up the server:

const express = require("express");

const app = express();

We can also do it all in a single line:

const app = require("express")();

To finish the setting up, we have to make the app listen to a port:

app.listen(8080);

It has to be on the bottom of the document. If it is higher, it won't work.

And that's it! Our Express server is ready to serve an API or HTML files.

GET Requests:

If we try to go to localhost:8080 in our newly created Express server, we will get an error: "Cannot GET /". This means that we have not added a Path to / and told Express what to show.

app.get("/", (req, res) => { res.send( { message: "our first route" } );});

Now, when we try to go back to localhost:8080/ we will see a JSON response: { message: "our first route" }.

So we learnt how to get information. But, is it possible to SEND information to the server via a GET request?

Short answer is yes. But we cannot do it from the Body. We have to use the URL. And there are two ways of doing it: Parameters or Queries.

Parameters are sent via the URL, and are represented in the server with a colon.

app.get("/bottle/:bottleSize", (req, res) => {

res.send({ theBottleIs: req.params.bottleSize })

})

Now, if we go to localhost:8080/bottle/large we are telling the server that the bottleSize param is "large". So, we can access the dynamic text that a user inputs and work with it in the back end. Unlike Queries, params are required.

Queries are also sent via the URL, and are represented with the question mark in the URL. It is not represented in the backend.

app.get("/bat", (req, res) => {

res.send({message: `The bat is ${req.query.adjective}`});

})

If we go to localhost:8080/bat?adjective=spooky we are sending information to the Backend. It gets it via the URL and we can manipulate it.
