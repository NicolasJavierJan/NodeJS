Client Side Redirection vs Server Side Redirection:

Client side redirect is a direct forward to a destination url. The redirect is induced by the client (browswer) directly:

location.href = "url"

location.replace("url")

etc...

Server Side Redirect happens when an HTTP request to a web server is answered with a server status code, which indicates that the requested document has moved to another URL. The client will then request the destination URL and the user gets forwarded.

Client Side Fetch vs Fetch in the FrontEnd

Client side fetching and front-end fetching are two related but distinct concepts in web development.

Front-end fetching encompasses all the activities related to retrieving data or resources on the client side, including Client Side Fetching, server side rendering, static site generation and client side caching.

Client-side fetching refers to the process of fetching data or resources from a server using JavaScript on the client side (like the browser) instead of relying on server-side code to generate the HTML page. Generally, client-side fetching uses the Fetch API to make requests to the server for data.
