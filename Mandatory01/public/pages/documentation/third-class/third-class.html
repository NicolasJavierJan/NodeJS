Loops and Loop Methods:

There are many, many ways to loop in JavaScript. Because there are many ways, there are also good and bad choiced when you need to loop.

A classic for loop

(for int i = 0; i < someNumber; i++)

will parse all of the items of a collection. In some ocasions, we need precisely this. But sometimes we don't, and it's a waste of time and resources to do it.

Map iterates, calls a callback function once for each element and constructs a new array from the results. It will normally not change the original array, but there are cases where it does, so we have to be very careful as to check that it did not modify the original as well.

Filter creates a copy of a portion of the given array, filtered down to the elements that pass the test implemented by the provided function.

Find will return the first element in the collection that satisfies the provided testing function.

Sort will take the array, convert it's elements to strings, comparing their sequences of UTF-16 code units values with each other, and sort it in ascending order.

Foreach will work similarly to map, but we use it in a different way. We would normally use the forEach function when we want to create HTML elements from the items in a collection, for example to populate a list or a table with data.

As we can see, there are many built-in iteration methods that we can use, each one with a different use that can help us be more efficient with time and resources.

Serving HTML in Express:

We can serve HTML directly in Express:

app.get("/about", (req, res) => {

res.send(`

<`h1>About<`/h1>

<`h3>This is my about page...!<`/h3>

`)

})

But this is strongly disencouraged, and extremely bothersome to do.

We can also do it in a more organized way: Have all the HTML, CSS and JS in a public folder, and serve that folder to Express.

app.use(express.static(path.resolve("public")));

By doing this, you can now load the files that are in the public directory by writing in the URL.
